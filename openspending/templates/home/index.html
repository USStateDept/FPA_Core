There are currently three components to FIND listed below. These will all be tied together once we have the design worked out.

Data Model Management

This has the database model in lists and fields. It is used to understand how each of the models are associated to each other.

Data Loading

This is used to load data into the system from various resources (URL or file upload).

Dataviewer (Data API URL generator)

This is used to develop the API calls, filter, and preview the data. When the link for the data is generated, you can use it in the FPA viz section to create visualizations. See info below on on the visualization section.

Data Visualization

This is used to visualize data from links or any other CSV format. Each indicator will have a page that automatically loads certain visualizations using this section. The visualizations will be loaded from stored JSON settings, as shown beside the "Fetch Options" button. Users with manager/admin rights will be able to extend these views to customize them for each indicator. Regular users (anonymous and registered) will be able to extend and it will be saved in a different section to become a "User Story" or "Data story".
The main component is that JSON can be saved with all of the settings of the user for the visualization to be recreated, customized, and saved in the database for future use.
